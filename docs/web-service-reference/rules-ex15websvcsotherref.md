# <a name="rules"></a>Rules

**L’élément Rules** contient un tableau de règles de protection. 
  
```xml
<Rules>   <Rule/></Rules>
```

 **ArrayOfProtectionRulesType**
## <a name="attributes-and-elements"></a>Attributs et éléments

Les sections suivantes décrivent les attributs, les éléments enfants et les éléments parents.
  
### <a name="attributes"></a>Attributs

Aucune.
  
### <a name="child-elements"></a>Éléments enfants

|**Élément**|**Description**|
|:-----|:-----|
|[Règle](rule.md) <br/> |Contient une règle de protection unique. Cet élément peut se produire zéro ou plusieurs fois. Cet élément se produit zéro fois lorsqu’aucune règle de protection n’est définie par l’organisation. Elle se produit une ou plusieurs fois si au moins une règle est définie par l’organisation.  <br/> |
   
### <a name="parent-elements"></a>Éléments parents

|**Élément**|**Description**|
|:-----|:-----|
|[ProtectionRulesConfiguration](protectionrulesconfiguration.md) <br/> |Contient la configuration du service pour le service de règles de protection.  <br/> |
   
## <a name="remarks"></a>Remarques

Le schéma qui décrit cet élément se trouve dans le répertoire virtuel EWS de l’ordinateur exécutant Microsoft Exchange Server 2010 sur qui le rôle serveur d’accès au client est installé.
